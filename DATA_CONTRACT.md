# Data Contract

This document defines which files belong to the **system** (auto-updatable) and which belong to the **user** (never touched by updates).

## User Layer (NEVER auto-updated)

These files contain your personal data, customizations, and work product. Updates will NEVER modify or overwrite them.

| File / Folder | Purpose |
|------|---------|
| `cv.md` | Your CV in Markdown |
| `config/profile.yml` | Your identity, targets, comp range |
| `modes/_profile.md` | Your archetypes, narrative, negotiation scripts |
| `data/applications.md` | Your application tracker database |
| `reports/*` | Your generated evaluation reports |
| `output/*` | Your compiled tailored PDF resumes |
| `jds/*` | Your saved job descriptions |
| `.env` | Your local API keys and secrets |
| `resume_data.json` | Temporary resume JSON file generated by Gemini |

## System Layer (safe to auto-update)

These files contain system logic, scripts, templates, and instructions that improve with each release.

| File / Folder | Purpose |
|------|---------|
| `modes/_shared.md` | Scoring system, global rules, tools |
| `modes/oferta.md` | Evaluation mode instructions |
| `src/*` | Clojure source files for headhunter-agent |
| `bb.edn` | Babashka task configuration |
| `resume.typ` | Base Typst file for compiling PDFs |
| `resume_template.typ` | Layout and styles template for Typst resumes |
| `setup.sh` | Unix setup script |
| `setup.bat` | Windows setup script |
| `README.md` | System-owned documentation |
| `LICENSE` | MIT License |
| `DATA_CONTRACT.md` | This file |

## The Rule

**If a file is in the User Layer, no update process may read, modify, or delete it.**

**If a file is in the System Layer, it can be safely replaced with the latest version from the upstream repo.**
